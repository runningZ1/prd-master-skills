# PRD Master - 生产级PRD生成器 v3.0

## 📖 Skill 简介

PRD Master 是**生产级PRD生成工具**，智能检测用户需求，提供两种模式：通用PRD生成、TaskMaster完整工作流。支持从概念到执行的完整产品需求文档生命周期，可直接用于生产环境。

**一个Skill，解决所有PRD需求！**

## 🎯 适用场景

### 🔧 通用生成模式
- **快速生成PRD**：使用5步工作流生成标准13部分PRD
- **创建用户故事**：掌握用户故事格式和验收标准编写
- **定义成功指标**：使用AARRR、HEART、OKRs等框架

### 🚀 TaskMaster模式
- **端到端工作流**：生成PRD并自动分解为可执行任务
- **自动化验证**：13项质量检查确保PRD完整性
- **任务执行**：4种执行模式（顺序/并行/完全自主/手动控制）

## 🌟 核心能力

### 0. 智能模式检测 ✨
**自动识别用户需求，无需手动选择模式**

- **智能分析**：根据输入关键词自动判断所需模式
- **两种模式**：🔧通用生成 / 🚀TaskMaster
- **渐进式引导**：根据需求逐步暴露功能
- **无缝切换**：支持模式间平滑过渡

**示例**：
```
用户："为用户认证创建PRD" → 自动进入🔧通用生成模式
用户："生成PRD并用TaskMaster执行" → 自动进入🚀TaskMaster模式
```

### 1. PRD 核心思路
- 文档目的：合适的方案 + 表达清晰
- 总分结构，符合金字塔原理
- "为什么要做" + "要做什么" 两大部分

### 2. 整体设计技巧
- 泳道流程图：多角色业务流程
- 实体关系图：系统模块关系
- 时序图：系统间交互顺序
- 状态机图：状态变化逻辑

### 3. 需求描述最佳实践
- "条件-动作-预期"三段式表达
- 字段说明表格化
- 时序图展示系统交互

### 4. 技术文档编写
- 系统架构设计
- 技术选型依据
- 接口定义和数据模型
- 与需求文档的双向追溯

### 5. 文档协作机制
- 统一的ID体系（REQ-XXX, TECH-XXX, API-XXX）
- 版本规范（v主版本.次版本.修订号）
- 评审机制（需求评审、技术方案评审、文档审核）

### 6. 高质量技术文章写作
- 八个核心方法：同理心、攒素材、定好题、搭骨架、画好图、有干货、勤润色、练内功
- 推荐工具和最佳实践

## 🔑 关键要点

1. **PRD = 合适的方案 + 表达清晰**
2. **使用总分结构，符合金字塔原理**
3. **根据需求特点选择合适的图表工具**
4. **建立双向追溯机制（需求 ↔ 技术）**
5. **使用"条件-动作-预期"三段式表达**
6. **保持文档同步更新和版本一致**
7. **需求方必须参与技术方案评审**
8. **使用统一的命名和版本规范**
9. **持续改进和优化流程**
10. **注重文档质量评估和团队能力建设**

## 📊 文档质量评估

使用以下维度评估文档质量：

- **完整性**：是否覆盖所有需求点
- **准确性**：描述是否准确无歧义
- **一致性**：需求和技术文档是否一致
- **可追溯性**：是否可以双向追溯
- **时效性**：文档是否及时更新
